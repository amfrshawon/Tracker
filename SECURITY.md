# Security Policy

## Supported Versions

This project is pre-1.0 and under active development.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

- Email: security@your-domain.com (replace this)
- Include reproduction steps, impact, and affected version/commit

## Security Practices

- Never commit `.env` or credential files
- Rotate any leaked key immediately
- Use least-privilege OAuth credentials
